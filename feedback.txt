### Phase 2 Implementation Feedback (Final Polish Suggestions)

#### 1. VisionInterface Symmetry
Currently, `main.py` uses `getattr(vision, "conf_threshold", ...)` because `conf_threshold` is not part of the `VisionInterface` abstract class.
*   **Suggestion:** Add `@property def conf_threshold(self) -> float:` and `def set_conf_threshold(self, value: float) -> None:` to `VisionInterface` in `vision/interface.py`. Implement them in `MockVision` as well. This enforces a consistent API across all vision implementations (Mock, YOLO, TFLite).

#### 2. Synchronization of Tests with Config
In `tests/test_fusion.py`, confidence values like `0.39` and `0.4` are hardcoded.
*   **Suggestion:** Replace these with `config.MIN_CONFIDENCE - 0.01` and `config.MIN_CONFIDENCE`. This ensures tests stay valid even if the default sensitivity is tuned in `config.py`.

#### 3. Moving Average Filter Reset on Failure
When a sensor failure occurs and the worker thread re-initializes the hardware, the old moving average buffer might contain stale or outlier values that caused the failure.
*   **Suggestion:** Add a `reset_filter()` method to `FusionEngine` (which calls `self._filter.reset()`). In `main.py`, if the sensor worker encounters an exception and re-initializes, consider calling this reset to ensure the moving average starts fresh with new, valid hardware data.

#### 4. CPU Temperature Mocking for Logic Readiness
`CsvLogger` already has a `cpu_temp` field, but it is currently always `0.0`. 
*   **Suggestion:** While real temperature reading is a Phase 4 task, adding a `MockTempSensor` or a simple random generator in `main.py` (when `RASEYES_MOCK` is active) would allow you to test the "Graceful Degradation" logic (from Phase 5) much earlier in the development cycle.

#### 5. Verification of Model Release in Benchmarks
Ensure `tests/benchmark_vision.py` is updated to confirm that `detector.stop()` successfully clears the model from VRAM/Memory, especially when running multiple benchmark passes in a single session.
