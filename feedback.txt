### 🛡️ Orange Pi 5 실장 테스트 및 이슈 해결 결과 (2026-07-07)

1. **Piper 한국어 모델 다운로드 URL 수정 완료**
   - 기존 `rhasspy/piper-voices`에 한국어 공식 모델이 존재하지 않아 다운로드에 실패했던 현상을 발견하고, KSS 한국어 데이터셋으로 학습된 커뮤니티 유지보수 모델인 `neurlang/piper-onnx-kss-korean` 경로로 다운로드 스크립트([scripts/download_piper_model.sh](file:///home/dong/WorkSpace/RasEyes/scripts/download_piper_model.sh))를 수정하여 올바르게 60MB의 ONNX 모델을 내려받아 기기에 전송 완료하였습니다.

2. **'pygoruut' PhonemeType 지원 몽키 패치 적용**
   - 한국어 모델의 `phoneme_type`이 `pygoruut`로 정의되어 있어 `piper-tts` 로딩 시 `'pygoruut' is not a valid PhonemeType` ValueError로 인한 EspeakTts Fallback 현상이 확인되었습니다.
   - [audio/piper_tts.py](file:///home/dong/WorkSpace/RasEyes/audio/piper_tts.py) 모듈 로딩 시점에 `piper.config.PhonemeType` 및 `piper.voice.PhonemeType` Enum을 동적으로 확장하고, `pygoruut` 패키지를 lazy import하여 음소 변환을 수행하도록 **몽키 패치**를 적용하여 이를 해결하였습니다.

3. **Piper TTS API 대칭성 확보 (1.4+ 및 Mock 호환)**
   - 기기에 설치된 `piper-tts 1.4.2` 버전에는 `synthesize_stream_raw` 메서드가 제거되고 `synthesize` 메서드로 변경되어 `AttributeError`가 발생하는 문제를 수정하였습니다.
   - `synthesize_stream_raw`의 유무를 체크하여 이전 버전 및 테스트의 Mock 동작을 지원하는 동시에, 새로운 `synthesize` API(Iterable[AudioChunk])로부터 `audio_int16_bytes`를 안전하게 추출할 수 있도록 동적 폴백 처리를 완벽하게 구성했습니다. (로컬 단위 테스트 `134 passed` 유지)

4. **실행 프로세스 갱신 및 서비스 상태 확인**
   - 수정된 코드를 안전하게 scp를 통해 Orange Pi에 배포한 후, systemd를 통하지 않고 유저 권한 내에서 `main.py` 프로세스를 킬하여 자동 재시작을 유도했습니다.
   - 서비스 재시작 로그 확인 결과, **`PiperTts 초기화 완료`** 및 **`부팅 오디오 큐 재생`** 로그까지 에러 없이 완벽히 구동됨을 확인했습니다.

### ⚠️ 추가 조치 권장 사항 (GPIO 권한 관련)

- 서비스 실행 로그에서 `ButtonHandler GPIO 초기화 실패: [Errno 2] No such file or directory` 경고가 발생하고 있습니다.
- 이는 `/dev/gpiochip1` 장치 파일에 일반 `orangepi` 유저 권한으로 접근할 수 없기 때문에 발생하는 현상입니다.
- **해결 방안:** Orange Pi 터미널에서 다음 udev 규칙을 추가하고 udevadm을 리로드하면 GPIO 버튼 기능이 정상화됩니다.
  ```bash
  # /etc/udev/rules.d/99-gpio.rules 파일 생성 및 아래 내용 기재
  KERNEL=="gpiochip*", MODE="0660", GROUP="dialout"
  
  # 규칙 리로드
  sudo udevadm control --reload-rules && sudo udevadm trigger
  ```
