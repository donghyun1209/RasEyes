## ToDo (다음 작업)

> 결론: 와이파이 동글 + rsync 로그 동기화로 진행. 블루투스는 느리고 구현 부담이 커서 제외. 영상은 상시 녹화 대신 경고 이벤트 전후 클립만 저장.

- [ ] **와이파이 동글 구매** — 사기 전에 칩셋 확인 필수. 리눅스 커널에 드라이버가 기본 포함된 칩셋(예: RTL8821CU, MT7601U 계열)으로 고를 것. 초저가 동글은 ARM 보드에서 드라이버 직접 빌드가 필요한 칩이 걸릴 수 있음.
- [ ] **동글 연결 및 확인** — Orange Pi 5에 꽂고 `lsusb`, `ip link`로 인식 확인 → NetworkManager로 집 와이파이 자동 연결 설정 → 무선 `ssh raseyes` 동작 확인.
- [ ] **로그 동기화 구성** — 1단계: PC에서 수동 `rsync raseyes:~/RasEyes/logs/ ./logs_archive/`. 2단계(선택): Pi에 systemd timer로 "네트워크 연결 시 PC로 rsync push" 자동화 (스크립트 10줄 이내, 전용 데몬 금지).
- [ ] **경고 이벤트 클립 저장 설계** — 상시 녹화 대신 링 버퍼 방식으로 경고 발생 시점 전후 몇 초만 저장. 저장소 수명·성능 KPI(500ms/15FPS)·프라이버시 문제를 피하기 위함. 구현 전 프레임 버퍼 구조와 저장 트리거 조건부터 설계.
